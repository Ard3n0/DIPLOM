# Архитектура данных и стратегия обучения

В проекте реализована концепция разделения статических эталонных знаний и динамического опыта. Это инженерное решение позволяет системе эффективно работать в условиях дефицита открытых корпусов математических текстов на русском языке.

## 1. Синтетическая генерация (Инициализация)

**Инструмент**: `math_generator.py`

Для первичного обучения классификатора применен метод процедурной генерации данных на основе декартова произведения сущностей.

### Механика и логика
Скрипт использует заранее подготовленные пулы данных: имена ученых, математические термины и шаблоны формулировок теорем. Это позволяет создать плотное признаковое пространство еще до получения первого пользовательского запроса.

### Разметка и балансировка
* **BIO-теги**: Система автоматически формирует разметку для каждого токена (Begin, Inside, Outside), подготавливая данные для задачи Token Classification.
* **Устранение дисбаланса**: Генератор обеспечивает репрезентативное количество примеров для всех 7 классов: O, B-TERM, I-TERM, B-NAME, I-NAME, B-FORMULA, I-FORMULA. Без этого этапа нейросеть будет игнорировать редкие сущности.

## 2. Автономное накопление данных (Continuous Training)

**Источник**: Пользовательский ввод через API

Система спроектирована как саморазвивающийся цикл (Self-supervised learning pipeline), функционирующий в процессе эксплуатации:

### Сбор (Shadow Data Collection)
Каждое предсказание модели проходит через фильтр уверенности. Если показатель `confidence` превышает установленный порог 0.50, токены и предсказанные метки асинхронно записываются в файл `autonomous_dataset.jsonl`.

### Триггер дообучения
При накоплении 50 новых записей (`RETRAIN_LIMIT`) система автоматически инициирует фоновую задачу переобучения. Это позволяет модели адаптироваться к специфическому стилю ввода конкретных пользователей.

### Изоляция и безопасность
Результаты дообучения сохраняются в отдельные директории с временными метками (`model_checkpoint_...`). Это предотвращает «отравление» базовой модели (Data Poisoning) и позволяет выполнить откат к эталонным весам в случае деградации точности.

## 3. Формат хранения и обработка

### JSONL (JSON Lines)
Формат выбран для обеспечения быстрой потоковой записи. В отличие от классического JSON, этот формат позволяет добавлять записи в конец файла без необходимости считывать весь массив в оперативную память. Это критически важно для производительности на системах с ограниченными ресурсами.

### Маскирование сущностей
Перед подачей в модель применяется механизм `mask_entities`, заменяющий конкретные значения (числа, даты) на абстрактные токены. Это снижает вариативность данных и заставляет нейросеть искать структурные паттерны, а не заучивать конкретные цифры.
